Comments

One of the best decisions a programmer can make is taking notes. Sure you will get to a point where you can understand the fundamental meaning of the code, but do you remember the bug related to it? Do you remember if that line of code affects other parts of code? Do you remember why you put it there in the first place? That's why we create comments.

/*
Block text
*/

// Line comment
1 #include <iostream>
2 using namespace std;
3
4 /*Main functions returns the string Hello World to the terminal
5 @return terminates the function, if there are no errors*/
6 int main()
7 {
8 //Cout displays the code to the terminal for the user
6 cout << "Hello World!"\n;
7 return 0;
8 }

There are two types of comments that you can create, block text and line comments. Block texts can span multiple lines as long as they are within the forward slash and asterisks /*...*/. Line comments on the other hand can only be one line and are identified by the two forward slashes //. There main purpose is to help the programmer understand the code they're looking at.
