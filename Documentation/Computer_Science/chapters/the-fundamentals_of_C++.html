The Fundamentals of C++

This lecture serves as an introduction to C++ and overall programming. It focuses on terminology, structure, and step by step instructions for C++ coding. The only prerequisites for this lecture are to have a working computer and a fundamental understanding of how to use it. The first section goes over the programming environment, explaining the applications and tools needed to write and run code. The next section will analyze basic code to understand its structure, keywords, and symbols. The final section goes into detail about the whole process, from preparing the proper tools to executing your first program.

Knowing your Programming Environment

A programming environment is a set of tools that aids a developer in creating a operational program. The tools mention here are absolutely essential and will be explained in detail. These tools include a text editor, a terminal, and a compiler.

Referring to the first tool, a text editor is an application that helps users write and edit text. Luckily, all operating systems (OS)s come with one. For example, Microsoft Windows has Notepad, MacOS has TextEdit, and the popular Ubuntu - one of the many distributors of Linux OS - has Vim. Programmers are not limited to these three text editors. There are open-source text editors that can come with extra features, such as Microsoft's Visual Studio Code. The options can be overwhelming, which is why for learning purposes and uniformity, this lecture and following lectures will use Vim. The reason for using Vim is because it's compatibility with all operating systems, and it's open-source software. Vim is also an application that can be used in the terminal. The terminal is an important part of the programming environment.

The terminal is a text interface. It looks like a simple window with a couple of characters and a blinking cursor. All recent operating systems have a terminal. Unfortunately, the terminal by itself isn't useful, which is why there are shells and command-line tools. A shell is a program - also known as a command-line interpreter - that gives a user access to the operating system services by providing a list of commands the user can input in the terminal. There are many shells that can be used with the terminal. Some popular ones include Bash, PowerShell, and Command Prompt (cmd). Caution, the shell isn't a visible application, it works in the background only giving hints of its existence through the terminal. Together, they allow users to access command-line applications (like Vim), manipulate files, install other tools, and other services a programmer needs.

For learning purposes, this lecture and future lectures will use the terminal along with bash. Both linux OS and MacOS come with Bash shells in the terminal. Windows users need to take an extra step by installing Windows Subsystem for Linux (WSL) using the guide on the Microsoft website. Windows Subsystem for linux creates a Linux environment on a Windows system, allowing user to execute Bash commands on the terminal and more. For those who don't want to install WSL, it is possible to use cmd and Notepad, but additional research is required. With the terminal and Bash, it is possible to write and store code. Though, to have the program run the terminal needs to install a compiler.

A compiler is software that translates high-level language (like C++) into low level language (such as assembly) for the system to understand. One of the most popular compilers is the GNU C++ Compiler - also known as g++ - and is what this lecture will use to help translate code. It is easy to install one within the terminal, and later sections will explain the installation process. Having to do all these steps can be a hassle, fortunately for some there are applications that simplify this process, allowing them to code right away.

Integrated development environments (IDE)s are applications that come with the same equipment that editors and terminals have, and then some. Warning, some of them state that they are free to use under certain conditions, like Microsoft's Visual Studio - not to get confused with Microsoft's Visual Studio Code which is just the editor. Basically, the software can only be used for studying purposes, and programmers can't make a profit unless they buy the business versions. While text editors and terminals are open source (FREE!). Companies like free, people like free, so it's best to follow the practices. Before writing any code, it is essential to understand what is written.

The Basic Structure of Any Code

Here is a simple HelloWorld program which contains a couple lines of code. It provides the fundamental structure of how every C++ program is built. C++ is case-sensitive. For example, the key term return is different from the word Return. It also has a free-form layout, so space and line breaks are ignored during compilation. Caution, compiler doesn't ignore the content between line breaks. For example, "s t d" is interpreted as the letters s, t, and d while "std" is interpreted as the keyword std. Still, a programmer can write each symbol, keyword, integer, and string - a collection of characters encased with double quotation marks ("") - on their own line, and it will still compile like the code below. The following will explain each line in great detail.

1 #include <iostream>
2
3 using namespace std;
4
5 int main()
6 {
7 cout << "Hello World!" << "\n";
8 return 0;
9 }

#include <iostream>

On line 1, there is the #include directive, which tells the compiler to look at the contents of the iostream header file that is within the less-than operator (<) and greater-than operator (>). The iostream file contains a list of input and output commands to query the user for information, or to display information to the user. This file will be examined in more detail in the stream manipulation lecture. Note, to use multiple header files in C++, each one needs a #include directives which can span multiple lines in the beginning of a program.

using namespace std;

Line 3 tells the compiler to use the standard name space to identify keywords in the programmer's code that are a part of header file's list of commands. An example would be the keyword cout, since it is a part of the iostream's list of commands. It is possible to override the functionality of key terms in the program by calling a different name space, but due to learning circumstances, these lectures will use the standard.

int main()

Let's look at the following statement on line 5. The main() is a function for the program. Every program has a main function and can have additional functions. Each function carries out a process, which consist of code. The parentheses are where we can store parameters - additional instructions for the function to use - though since this is a simple program, it is left empty. Functions will be explained in future lectures. Finally, the keyword int is called a data type, this specific data type says that the main function can return an integer.

{ ... }

Both lines 6 and 9 help identify where the code of the main function begins and ends. This is one of the many locations where programmers will write their code.

cout << "Hello World!"\n;

The code above uses the keyword cout, followed by two less-than operators (<<) to send anything on the right side of the operators to the terminal. The part "Hello World!" is the string that is sent and displayed in the terminal. Remember, a string is a set of characters surrounded by double quotation marks (""). The double quotation marks help the compiler distinguish between code and text for the user to view. The escape sequence, newline (\n), is one character according to the compiler. It tells the compiler to break from the existing line, and to start on a new line in the terminal. The semicolon (;) is like a period and symbolizes the end of a statement.

return 0;

Remember how I said the main function expects an integer to be returned, well this is it. On line 8, the compiler says to return 0. The digit 0 is considered a flag, it signals the compiler that there were no errors in the program, and that it can be terminated. However, if there are errors, the terminal will provide a long list of what seems like random words at first, but it is actually important information to fix the code.

Errors

There are two types of errors in programming, syntax errors and logical errors. A syntax error is a compilation error that detects missing symbols, values, and keywords from the program. An example would be when a programmer forgets the semicolon or forgets an operator. This is usually display in the terminal, stating the file and the location of the error in that file. A logical error happens during the execution of the program and is only caught by the user. It can be anything from the code misbehaving and giving wrong values to not give values at all. Compilers can't find these errors, but a debugger can help find where the issue is at.

Preparing and Understanding the Terminal

In this section I assume that you have your terminal with Bash opened and are ready for the next instructions. This lecture will try to anticipate any issues, but for further assistance please search online. For now, we will be going over basic commands, updating the terminal, installing the compiler, and running code.

Commands

Before writing a program, this section will go over fundamental commands and how to traverse the directory. Consider a directory as tree-like collection of files, where each directory branches to subdirectories (files within files), and where the root of the tree is usually the storage location.

pwd Command

$ pwd


Provides a path to the currently accessed directory. A path consist of a root directory, and all subsequent directories separated by a forward slash (/). Note, in Windows OS each step of the path is separated by a backslash (\). Remember this when referring to Linux or Windows, since getting the two mixed up can cause errors.

ls Command

$ ls


Provides a preview of the subdirectories in the current directory.

cd Command

$ cd directory_name
$ cd ..
$ cd


Helps navigate the directory and can require additional input. For instance, to step into a subdirectory, type and enter the subdirectory's name after the cd command. To step back one directory, type two periods after the cd command. To step all the way back to the origin of the directory, enter cd by itself. Rather than using cd for every step, if the path of the desired directory is known, type the path after the command cd and then enter. Looking at the example, notice the path depicted in the green section, or notice the subdirectories when we enter the command ls.

mkdir Command

$ mkdir directory_name


Create a subdirectory in the current directory.

vim Command

$ vim file_name.cpp


Creates and accesses the user defined C++ file. Note, these files have a (.cpp) extension which can be interpreted as “C Plus Plus”. Vim has modes and commands to edit, manipulate, save, and more. Typing right away will cause the first few characters from the keyboard to not register. To start writing/editing text, press the character (i) on the keyboard. This is known as insert mode, and is shown at the bottom of the window. To quit any of the several modes, press the escape key. Deleting characters while not in insert mode will cause the cursor to move to the left instead of deleting the letters. To fix this, hit the (i) key again.

Vim has unique commands that can be entered in the command line, located at the bottom of the window. Before typing a command, exit out of any mode. Each command starts with a colon (:). To know more about Vim, enter (:help) on the command line. This lecture will go over a few of the basic commands. To get line numbering, enter (:set number) in the command line. For saving files, type (:w) in the command line and press enter. To quit Vim and go back to the terminal, use the following command (:q). Note, if a file has been edited and the quit command is executed before saving, Vim will warn the user about unsaved changes. To bypass this enter the force quit command (:q!). Warning, all progress will be lost, so the better option is to use the save and quit command (:wq).

Remember, Vim is an open-source editor. It comes with some distributors of Linux like Ubuntu, but if it's not installed, the terminal will complain and suggest installing it using whatever command it provides. Any editor can work with C++ files, but for educational purposes, this lecture will use Vim.

rmdir Command

$ rmdir directory_name


Removes a subdirectory in the current directory. Warning, only removes empty directories.

rm Command

$ rm file_name


Removes files that are in the current directory.

sudo Command

$ sudo ...


This command gives administrative access over operations. Any commands with sudo tagged onto it will initially ask for a password before doing its task, so make sure to remember the password. Knowing sudo, will allow users to update and install packages.

Updating

To avoid issues when installing the compiler check for updates. To do this, there are a set of instructions to follow. First, update the package list by entering the following command:

$ sudo apt-get update

The command "apt-get update" is part of the Advanced Package Tool (APT) that helps manage software. In this case, the command above is trying to get updates on the current packages to allow users to upgrade to the proper software. Packages is just another way to say applications or software. To upgrade, type the following in the terminal:

$ sudo apt-get upgrade

The last step ensures critical packages are upgraded, while it cleans up unnecessary packages. Type the following:

$ sudo apt-get dist-upgrade

If successful, everything should be up to date. The next step is installing the compiler

Installing Compiler

There are two methods to download the package, but for this instance, use the build-essential route since it includes other key features like: libraries, debuggers, and the compiler itself. To do this, type the following:

$ sudo apt install build-essential

Afterwards, check to see if it is installed by entering:

$ g++ --version


If you see something like the image above, then the compiler has been successfully installed. The next step is to write a program.

Writing, Compiling, and Running Code

To begin, set up a user defined directory to store programs, in this case call it "Programs".

$ mkdir Programs

After creating a directory, access it with the cd command.

$ cd Programs

Once inside, begin creating a program file. Use the vim command and call the file "HelloWorld.cpp".

$ vim HelloWorld.cpp

Once the text editor opens, remember to hit the (i) key to begin insert mode. Type the Hello World example from the beginning of the lecture. Once completed, save the file by getting out of insert mode and type the save quick command (:wq).

After exiting the editor, begin compiling the file. Like the terminal and the editor, g++ has its own unique commands. The first step is to translate the (.cpp) file into an object file; it's a file with the extension (.o) attached to it. To clarify, the object file contains machine level code, which is something the system can understand. To start the process type:

$ g++ -c file_name.cpp $ g++ -c HelloWorld.cpp

Once the object file is created it will be in the current directory. Use command (ls) to view the new file. Notice how it has the same name as the C++ file, this is normal. At this point a syntax error may show instead of a file. The error's text will identify the location of the error in the form of text in the terminal. Simply go back and edit the file. If successful, the next step is to name and create an executable file. For C++, the executable file has a (.out) extension. This file will run and provide output from the program. Use the following example to create it:

$ g++ -o executable_file_name file_name.cpp $ g++ -o HelloWorld HelloWorld.cpp

Typing the command (ls) will show the updated contents of the current directory. Notice how there is no extension with this file. To run it, use the dot-forward-slash (./) to denote the current directory with the name of the executable file attached ike so:

$ ./executable_file_name $ ./HelloWorld


Once submitted, the output will be displayed on the next line in the terminal. If it runs and there are issues with the output, then it is considered a logical error. To fix this, go back into the file and carefully look for the problem. Once fixed, use the last 3 steps to compile and execute the program again. Congratulations, you are officially a programmer!
